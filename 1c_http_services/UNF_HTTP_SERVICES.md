# Публикация HTTP-сервисов в 1С:УНФ

## 1. Создание HTTP-сервиса

1. Откройте конфигурацию в **Конфигураторе**.
2. В дереве метаданных найдите ветку **Общие → HTTP-сервисы**.
3. Нажмите **Добавить** → создайте HTTP-сервис с именем **api**.
4. В поле **Корневой URL** укажите **api** (или оставьте пустым, если хотите `/hs/api` — тогда укажите `api`).

---

## 2. URL-шаблоны и модули

Для каждого эндпоинта создайте URL-шаблон внутри HTTP-сервиса `api`.

### 2.1. GET `/hs/api/stock`

**URL-шаблон:** `stock`  
**Метод:** GET  

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| item | string | нет | Фильтр по номенклатуре (частичное совпадение) |
| organization | string | нет | Фильтр по складу (частичное совпадение) |
| min_quantity | int | нет | Минимальный остаток |
| nomenclature | string | нет | Синоним `item` |
| warehouse | string | нет | Синоним `organization` |

**Формат ответа:**
```json
[
  {
    "item": "Гвоздь 100мм",
    "organization": "ООО СтройМастер",
    "quantity": 1500
  }
]
```

**Код обработчика (вставить в модуль URL-шаблона):**

```1c
Функция getStock(Запрос) Экспорт

    Товар = ПолучитьПараметрHTTPЗапроса(Запрос, "item");
    Если ПустаяСтрока(Товар) Тогда
        Товар = ПолучитьПараметрHTTPЗапроса(Запрос, "nomenclature");
    КонецЕсли;

    Организация = ПолучитьПараметрHTTPЗапроса(Запрос, "organization");
    Если ПустаяСтрока(Организация) Тогда
        Организация = ПолучитьПараметрHTTPЗапроса(Запрос, "warehouse");
    КонецЕсли;

    МинКоличество = ПолучитьПараметрHTTPЗапроса(Запрос, "min_quantity");
    МинКоличествоЧисло = ?(ЗначениеЗаполнено(МинКоличество), Число(МинКоличество), 0);

    Выборка = РегистрыНакопления.ЗапасыНаСкладах.Выбрать(, );

    Остатки = Новый Соответствие;

    Попытка
        Пока Выборка.Следующий() Цикл

            // Фильтр по номенклатуре (частичное совпадение)
            Если ЗначениеЗаполнено(Товар) Тогда
                Если СтрНайти(ВРег(Выборка.Номенклатура), ВРег(Товар)) = 0 Тогда
                    Продолжить;
                КонецЕсли;
            КонецЕсли;

            // Фильтр по складу (частичное совпадение)
            // Используем измерение "Склад" регистра ЗапасыНаСкладах
            Если ЗначениеЗаполнено(Организация) Тогда
                Попытка
                    Если СтрНайти(ВРег(Выборка.Склад), ВРег(Организация)) = 0 Тогда
                        Продолжить;
                    КонецЕсли;
                Исключение
                    // Если поля "Склад" нет, пробуем "Организация"
                    Если СтрНайти(ВРег(Выборка.Организация), ВРег(Организация)) = 0 Тогда
                        Продолжить;
                    КонецЕсли;
                КонецПопытки;
            КонецЕсли;

            // Фильтр по минимальному количеству
            ИмяТовара = Строка(Выборка.Номенклатура);
            ИмяОрганизации = Строка(Выборка.Организация);
            Ключ = ИмяТовара + "|" + ИмяОрганизации;
            ТекКоличество = Выборка.Количество;

            Если Остатки[Ключ] = Неопределено Тогда
                Остатки[Ключ] = Новый Структура("item, organization, quantity", ИмяТовара, ИмяОрганизации, ТекКоличество);
            Иначе
                Остатки[Ключ].quantity = Остатки[Ключ].quantity + ТекКоличество;
            КонецЕсли;
        КонецЦикла;

        Массив = Новый Массив;
        Для Каждого КлючЗначение Из Остатки Цикл
            Если КлючЗначение.Значение.quantity = 0 Тогда
                Продолжить;
            КонецЕсли;
            Если МинКоличествоЧисло > 0 И КлючЗначение.Значение.quantity < МинКоличествоЧисло Тогда
                Продолжить;
            КонецЕсли;
            Массив.Добавить(КлючЗначение.Значение);
        КонецЦикла;

        Ответ = Новый HTTPСервисОтвет(200);
        Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
        Ответ.УстановитьТелоИзСтроки(ЗначениеВСтрокуВнутр2(Массив));
        Возврат Ответ;
    Исключение
        Ответ = Новый HTTPСервисОтвет(500);
        Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
        Ответ.УстановитьТелоИзСтроки("{""error"": """ + ОписаниеОшибки() + """}");
        Возврат Ответ;
    КонецПопытки;

КонецФункции
```

**Что изменилось в коде:**
- Фильтрация по частичному совпадению (`СтрНайти`) вместо точного (`<>`)
- Добавлен фильтр `min_quantity`
- Поддержка синонимов: `item`/`nomenclature`, `organization`/`warehouse`
- Обработка ошибок через `Попытка/Исключение`

### 2.2. GET `/hs/api/sales`

**URL-шаблон:** `sales`  
**Метод:** GET  

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| date_from | string (ISO) | нет | Начальная дата (YYYY-MM-DD) |
| date_to | string (ISO) | нет | Конечная дата (YYYY-MM-DD) |
| manager | string | нет | ФИО менеджера (частичное совпадение) |
| item | string | нет | Название товара (частичное совпадение) |

**Формат ответа:**
```json
[
  {
    "date": "2025-01-15",
    "item": "Гвоздь 100мм",
    "quantity": 200,
    "sum": 4000.0,
    "manager": "Иванов И.И."
  }
]
```

**Код обработчика:**

```1c
Функция ПродажиЗаПериод(Запрос) Экспорт

    ДатаНач = ПолучитьПараметрHTTPЗапроса(Запрос, "date_from");
    ДатаКон = ПолучитьПараметрHTTPЗапроса(Запрос, "date_to");
    Товар = ПолучитьПараметрHTTPЗапроса(Запрос, "item");
    Менеджер = ПолучитьПараметрHTTPЗапроса(Запрос, "manager");

    Попытка
        Выборка = РегистрыНакопления.Продажи.Выбрать(, , );

        Массив = Новый Массив;
        Пока Выборка.Следующий() Цикл

            // date_from: оставляем только записи >= начальной даты
            Если ЗначениеЗаполнено(ДатаНач) И Выборка.Период < Дата(СтрЗаменить(ДатаНач, "-", "")) Тогда
                Продолжить;
            КонецЕсли;

            // date_to: оставляем только записи <= конечной даты (+1 день, т.к. Период может содержать время)
            Если ЗначениеЗаполнено(ДатаКон) И Выборка.Период >= Дата(СтрЗаменить(ДатаКон, "-", "")) + 86400 Тогда
                Продолжить;
            КонецЕсли;

            // Фильтр по товару (частичное совпадение, регистронезависимо)
            Если ЗначениеЗаполнено(Товар) И СтрНайти(ВРег(Выборка.Номенклатура), ВРег(Товар)) = 0 Тогда
                Продолжить;
            КонецЕсли;

            // Фильтр по менеджеру (частичное совпадение)
            Если ЗначениеЗаполнено(Менеджер) И СтрНайти(ВРег(Выборка.Ответственный), ВРег(Менеджер)) = 0 Тогда
                Продолжить;
            КонецЕсли;

            Стр = Новый Структура;
            Стр.Вставить("date", Формат(Выборка.Период, "ДФ=yyyy-MM-dd"));
            Стр.Вставить("item", Строка(Выборка.Номенклатура));
            Стр.Вставить("quantity", Выборка.Количество);
            Стр.Вставить("sum", Выборка.Сумма);
            Стр.Вставить("manager", Строка(Выборка.Ответственный));
            Стр.Вставить("client", Строка(Выборка.Контрагент));  // 👈 добавить покупателя
            Массив.Добавить(Стр);

        КонецЦикла;

        Ответ = Новый HTTPСервисОтвет(200);
        Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
        Ответ.УстановитьТелоИзСтроки(ЗначениеВСтрокуВнутр2(Массив));
        Возврат Ответ;
    Исключение
        Ответ = Новый HTTPСервисОтвет(500);
        Ответ.Заголовки.Вставить("Content-Type", "application/json; charset=utf-8");
        Ответ.УстановитьТелоИзСтроки("{""error"": """ + ОписаниеОшибки() + """}");
        Возврат Ответ;
    КонецПопытки;

КонецФункции
```

**Что исправлено:**
- `date_to` — было `<` (как `date_from`), стало `>` — теперь работает как верхняя граница
- `item` — точное совпадение `<>` → частичное `СтрНайти(ВРег(), ВРег())`
- Добавлен фильтр `manager`
- Структура ответа: `item` вместо `nomenclature` (как и в stock)

## 3. Общая функция преобразования в JSON

Добавьте эту функцию в **общий модуль** (например, `HTTPСервисыВызовСервера`) или в каждый модуль URL-шаблона:

```1c
Функция ПреобразоватьВJSON(Данные) Экспорт

    ЗаписьJSON = Новый ЗаписьJSON;
    Результат = Новый Массив;

    // Определяем тип данных
    Если ТипЗнч(Данные) = Тип("Массив") Тогда
        ЗаписьJSON.УстановитьСтроку();
        ЗаписатьJSON(ЗаписьJSON, Данные);
        Возврат ЗаписьJSON.Закрыть();
    ИначеЕсли ТипЗнч(Данные) = Тип("Структура") Или ТипЗнч(Данные) = Тип("Соответствие") Тогда
        ЗаписьJSON.УстановитьСтроку();
        ЗаписатьJSON(ЗаписьJSON, Данные);
        Возврат ЗаписьJSON.Закрыть();
    Иначе
        ЗаписьJSON.УстановитьСтроку();
        ЗаписатьJSON(ЗаписьJSON, Данные);
        Возврат ЗаписьJSON.Закрыть();
    КонецЕсли;

КонецФункции
```

## 4. Настройка публикации на веб-сервере

### Apache
```apache
<Location /unf-kav>
    AuthType Basic
    AuthName "1C:Enterprise"
    AuthUserFile /etc/apache2/.htpasswd
    Require valid-user
</Location>
```

### IIS
1. Откройте IIS Manager.
2. Выберите сайт → **Authentication** → включите **Basic Authentication**.
3. Создайте пользователя с правами только на чтение.

### nginx
```nginx
location /unf-kav {
    auth_basic "1C:Enterprise";
    auth_basic_user_file /etc/nginx/.htpasswd;
    proxy_pass http://127.0.0.1:8080/unf-kav;
}
```

## 6. Известные проблемы и их решение

### 6.1. Поиск по кириллице не работает (nomenclature/search)

**Симптом:** GET `/hs/api/nomenclature/search?q=гвоздь` возвращает `[]`, хотя пустой поиск (`q=`) или поиск по цифрам работает.

**Причина:** IIS передаёт URL-параметры в 1С без корректной декодировки UTF-8. 1С получает параметр, но не может найти совпадение в базе.

**Решение (вариант А):** Добавить ручное декодирование в начале функции:

```1c
Функция searchNomenclature(Запрос) Экспорт

    Поиск = Запрос.Параметры["q"];

    // Ручное декодирование URL, т.к. IIS может не декодировать UTF-8
    Попытка
        Поиск = РаскодироватьСтроку(Поиск, СпособКодировкиСтроки.URLКодирование);
    Исключение
        // Если не получилось — оставляем как есть
    КонецПопытки;

    // ... остальной код
```

**Решение (вариант Б):** Создать отдельный POST-эндпоинт для поиска, где параметр `q` передаётся в теле запроса (JSON) — это гарантирует корректную передачу UTF-8.

### 6.2. Фильтрация stock/sales не работает

**Симптом:** Параметры `?warehouse=...&nomenclature=...` игнорируются, возвращаются все записи.

**Причина:** В `WHERE` отсутствуют скобки вокруг `ИЛИ`, из-за чего приоритет операторов неправильный:
```
// НЕПРАВИЛЬНО — AND имеет приоритет над OR
ГДЕ Количество > 0 И Склад = "" ИЛИ Склад ПОДОБНО &Склад

// ПРАВИЛЬНО
ГДЕ Количество > 0 И (Склад = "" ИЛИ Склад ПОДОБНО &Склад)
```

**Решение:** Использовать скобки как в исправленных примерах выше. После исправления переопубликуйте HTTP-сервис и проверьте:

```bash
curl -u "user:pass" "https://01.yes-it.ru/unf-kav/hs/api/stock?nomenclature=Гофрокороб"
```

### 6.3. В ответе stock/sales нет поля warehouse

**Симптом:** Ответ содержит `item`, `quantity`, но нет `warehouse`.

**Причина:** В 1С-коде используется JSON-ключ `item` вместо `nomenclature`, и пропущены поля `warehouse`/`unit`.

**Решение:** Проверьте, что в коде используются следующие ключи JSON (как в исправленных примерах):

| Поле в 1С | Ключ JSON | Описание |
|-----------|-----------|----------|
| Номенклатура | `nomenclature` | Название товара |
| Склад | `warehouse` | Название склада |
| Количество | `quantity` | Количество |
| Сумма | `sum` | Сумма продажи |
| Единица | `unit` | Единица измерения |

### 6.4. /receivables возвращает пустой массив

**Симптом:** GET `/hs/api/receivables` возвращает `[]`.

**Причина:**
- В базе действительно нет задолженностей (долгов клиентов)
- Или регистр `Взаиморасчеты` пуст/называется иначе

**Диагностика:** Проверьте в 1С, есть ли записи в регистре `Взаиморасчеты` с отрицательным остатком. Если регистр называется иначе (например, `ВзаиморасчетыСКлиентами`), замените имя в запросе.

## 7. Примечания по УНФ

- **Регистр "ПартииТоваров"** — содержит остатки по складам
- **Регистр "Продажи"** — содержит обороты по продажам
- **Регистр "Взаиморасчеты"** — содержит взаиморасчеты с контрагентами (отрицательный остаток = долг клиента)
- Если в вашей конфигурации имена регистров отличаются, замените их в запросах

## 8. Альтернатива: упрощённый вариант через OData

Если нет возможности публиковать HTTP-сервисы, можно временно использовать **OData**. Пример OData-запроса к справочнику номенклатуры:

```bash
curl -u "Андрей Кузнецов:250879" \
  "https://01.yes-it.ru/unf-kav/odata/standard.odata/Catalog_%D0%9D%D0%BE%D0%BC%D0%B5%D0%BD%D0%BA%D0%BB%D0%B0%D1%82%D1%83%D1%80%D0%B0?\$top=5&\$select=Ref,Description"
```

Но OData требует URL-кодирования русских имён и сложнее в парсинге. Рекомендуется публикация HTTP-сервисов.
