# HTTP-сервисы 1С для интеграции

## Общая информация

Для работы MCP-сервера необходимо опубликовать в конфигурации 1С:УТ 11 / УНФ следующие HTTP-сервисы.

## Эндпоинты

### 1. GET /api/stock

Остатки товаров на складах.

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| warehouse | string | нет | Фильтр по складу |
| nomenclature | string | нет | Фильтр по номенклатуре |
| min_quantity | int | нет | Минимальный остаток |

**Формат ответа:**
```json
[
  {
    "nomenclature": "Гвоздь 100мм",
    "warehouse": "Москва",
    "quantity": 1500,
    "unit": "шт"
  }
]
```

**Регистр учета:** Остатки товаров (РегистрНакопления.ОстаткиТоваров)

### 2. GET /api/sales

Данные о продажах.

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| date_from | string (ISO) | нет | Начальная дата |
| date_to | string (ISO) | нет | Конечная дата |
| manager | string | нет | ФИО менеджера |
| warehouse | string | нет | Название склада |

**Формат ответа:**
```json
[
  {
    "date": "2025-01-15",
    "nomenclature": "Гвоздь 100мм",
    "quantity": 200,
    "sum": 4000.0,
    "manager": "Иванов И.И.",
    "warehouse": "Москва"
  }
]
```

**Регистр учета:** Продажи (РегистрНакопления.Продажи)

### 3. GET /api/sales/by_manager

Агрегированные продажи в разрезе менеджеров.

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| date_from | string (ISO) | нет | Начальная дата |
| date_to | string (ISO) | нет | Конечная дата |
| manager | string | нет | ФИО менеджера |

**Формат ответа:**
```json
[
  {
    "manager": "Иванов И.И.",
    "total_sum": 45000.0,
    "total_quantity": 500
  }
]
```

### 4. GET /api/receivables

Задолженность клиентов.

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| min_amount | float | нет | Минимальная сумма долга |
| date_from | string (ISO) | нет | Начальная дата |

**Формат ответа:**
```json
[
  {
    "client": "ООО СтройМастер",
    "amount": 250000.0,
    "overdue_days": 45
  }
]
```

**Регистр учета:** Взаиморасчеты (РегистрНакопления.Взаиморасчеты)

### 5. GET /api/nomenclature/search

Поиск номенклатуры по названию.

**Параметры запроса (query):**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|-------------|----------|
| q | string | да | Поисковый запрос |
| limit | int | нет | Лимит результатов (по умолчанию 10) |

**Формат ответа:**
```json
[
  {
    "ref": "00000001",
    "name": "Гвоздь 100мм",
    "unit": "шт"
  }
]
```

**Справочник:** Номенклатура (Справочник.Номенклатура)

## Инструкция по публикации в 1С

1. Открыть конфигурацию в режиме Конфигуратор.
2. Создать новый HTTP-сервис через меню `Администрирование → Публикация веб-сервисов`.
3. В модулях HTTP-сервиса написать обработчики для каждого эндпоинта.
4. Настроить Basic Auth на уровне веб-сервера (IIS / Apache).
5. Проверить через curl:

```bash
curl -u service_user:service_password http://localhost/1c/api/stock
```

## Альтернатива: OData

Вместо кастомных HTTP-сервисов можно использовать стандартный OData-интерфейс 1С. Пример запроса:

```bash
curl -u service_user:service_password \
  "http://localhost/1c/odata/standard.odata/Register_%D0%9E%D1%81%D1%82%D0%B0%D1%82%D0%BA%D0%B8%D0%A2%D0%BE%D0%B2%D0%B0%D1%80%D0%BE%D0%B2"
```

Однако кастомные HTTP-сервисы предпочтительнее, так как дают:
- Более простой JSON (без вложенности OData)
- Контролируемую фильтрацию на стороне 1С
- Безопасность (только чтение, только нужные регистры)
